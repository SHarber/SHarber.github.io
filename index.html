Introduction to the Deck Of Cards API

What is the Deck of Cards API?

The Deck of Cards API is an API created to simulate a deck of cards. Card games only require a few simple actions to play any card game you can imagine. The deck of cards API was created to simulate those actions. You can use the deck of cards API to create any game you would like!

What you will learn in this guide:

  • What an API is and how to use them.
  • Specific uses of the Deck of Cards API.
  • How to create a simple Blackjack game

What is an API?

API stands for Application Programming Interface.

The most important thing to remember about APIs is the Interface (the I in API). Interfaces simplify complicated systems and tasks for the user. Think of API’s like learning to drive a car. When you learn how to drive, you don’t need to know how to build a car, how the engine works, or how every part makes the car drive. You just need to know how to use the interface: the pedals, the steering wheel, the mirrors.

APIs allow for a standarization of information and communication.

Specifically, a web API uses URLs as controls. You request information from the URL and it sends back information as a response. These responses are usually plain text with the requested information. In web development, we use this information to create a webpage that looks nice for the user (you).

Deck of Cards API

The Deck of Cards API simulates a deck of cards with all the possible actions needed to create any card game you desire! In this how-to guide, we are going to create a simple black jack game!

Let's see What Deck Of Cards Can Do!

Get a Deck of Cards

The first step to using the deck of cards API is to get a deck ID. The Deck Id is used to keep playing on the same deck you started the game with. The API allows you to get a deck ID using 3 of their APIs.

  • "A Brand New Deck"
  • "A Partial Deck"
  • "Shuffle The Cards"

Let’s look at these in a little more detail.

A Brand New Deck does exactly what you would think it does. Imagine opening up a brand new deck of cards from the store. There are 52 cards (minus jokers) in a beautiful nice order. (Ace of spades, 2 of spades etc.)

A Partial Deck is used when you want only certain cards in your deck. When calling this API you will get a deck back with only the individual cards you have chosen. This would be ideal for creating games like Euchre.

Finally, we have Shuffle The Cards! This allows us to create a new deck of cards, specify the number of decks we would like to use, and it shuffles all of the decks together. This is ideal for a black jack game, because when playing black jack 6 decks of cards are used.

So let's get our deck!

“A Brand New Deck”, “A Partial Deck”, and “Shuffle The Cards” will all return 4 things to us after we request our new deck:

  1. If the deck is shuffled (‘shuffled’).
  2. How many cards are remaining in the deck (‘remaining’).
  3. Our deck id (‘deck_id’).
  4. If our request was successful (‘success’).

We are going to focus on “Shuffle the Cards” since this is the API needed for the 6 decks used in black jack.

The URL needed to make a “Shuffle the Cards” request is:

https://deckofcardsapi.com/api/deck/new/shuffle/?deck_count=6

The deck_count in the url is where you specify how many decks you would like to use. In this case, we are using 6 decks.

Making an AJAX Request

Now that we know what the API does, and we have the URL, we need to know how to request the information and get a response back. This is done using an AJAX request in our JavaScript code.

What is AJAX?

AJAX stands for Asynchronous JavaScript and XML. In the past, XML (Extensible Markup Language) was the only choice for open data interchange, however, over the years JSON (JavaScript Object Notation) has become more popular as its less verbose and faster. Our examples will use JSON.

Now Let's look at some code!

The first step is to create a new XMLHttpRequest. Although we are using JSON instead of XML we still use this as it is compatible with a JSON request.

Next we are going to initialize the object through an open request. This must occur before sending the request. In our open request, we will state what type of request (GET or POST), the API URL needed to make the request, and if you want to make a synchronous request(false) or an asynchronous request (true).

To use Deck of Cards API you can use either a GET or a POST request. We will only be using GET and asynchronous requests. ***Note: The Request for a Deck ID MUST be complete before any other APIs are used*** Once we submit our request, we will get a response back that looks like this:

Drawing Cards

Deck of Cards has an API Called “Draw a Card”. This is how we will deal our cards to the player and the dealer. We will make another AJAX request using JSON.

The URL for a “Draw a Card” request is:

https://deckofcardsapi.com/api/deck/deck_id/draw/?count=2

The key to this request is to replace deck_id with the deck id we created in the last step. The count is the number of cards you want to draw. Let’s create a request to draw 2 cards for the player, and 2 cards for the dealer. Here is an example of our request:

”Draw a Card” Response

As you can see in our example above, we requested 2 cards from our deck using our deck ID. The response from this request returns a "cards" object. The cards object will contain the number of cards you requested in the form of more objects. The individual sub-card objects will contain:

  • Card Code: The card code is composed of either 2 letters, or a number and a letter. This is a unique identifier for each card. For example, 9D stands for the 9 of diamonds, QS stands for the queen of spades, etc.
  • URL for an image of the card. The .png image is returned under the image element. This response also returns an images object which contains a .png and .svg image file
  • The suit of the card.
  • The value of the card.

This will give you all of the information you will need to create an interface and rules for your own blackjack game.
